# resource-home-main

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态资源的形式存放，不依赖特定的后端环境，可直接通过浏览器打开或部署到任意静态托管服务。

仓库内容为通用型 HTML 页面集合，不针对任何单一域名或具体网站。每个页面均独立存在，互不依赖，便于检索、预览和复用。

## 目录说明

```
resource-home-main/
├── index.html          # 仓库根入口页面（可选）
├── pages/              # 存放各类独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 页面引用的公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面存放于此，每个文件为一个完整的页面。
- `assets/`：存放页面共享的静态资源，按类型组织子目录。

## 页面归档说明

- 每个 HTML 页面均为自包含的文档，可直接运行。
- 页面可引用 `assets/` 下的公共资源，也可内联样式和脚本。
- 页面命名应简洁明了，便于识别内容类别。
- 若页面包含外部依赖（如字体、图标库），建议使用 CDN 链接或自行托管至 `assets/`。

## 使用方式

1. 克隆本仓库到本地：
   ```
   git clone https://github.com/your-username/resource-home-main.git
   ```
2. 直接双击 `pages/` 下的 HTML 文件，或通过本地服务器预览（推荐）：
   ```
   cd resource-home-main
   python -m http.server 8000
   ```
3. 也可将整个仓库部署至任意静态托管平台。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 `.html` 文件，如需资源可放入 `assets/` 对应目录。
- 修改页面：直接编辑对应 HTML 文件，注意保持引用路径正确。
- 删除页面：移除文件及不再使用的相关资源。
- 建议每次变更后更新本 README（如新增页面列表），但非强制。

## 许可

本仓库内容仅供学习和参考使用。部分页面可能包含第三方资源，请遵循其各自许可协议。
